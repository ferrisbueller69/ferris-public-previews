Heads up — read this

  • Web NFC works on Android Chrome only. iOS Safari does not expose NFC to websites; iPhone users must paste the UID manually (or scan via a native iOS app using Core NFC).
  • This pass is unsigned. A real iPhone will refuse to add it to Wallet. To install on a device you need an Apple Developer account, a Pass Type ID, the WWDR cert, and a server-side signing step (CMS detached signature over manifest.json).
  • Apple Pay payment cards cannot be provisioned from a website. That is restricted to issuing banks via Apple's PassKit issuer program. Event tickets / loyalty / coupons are fine.
